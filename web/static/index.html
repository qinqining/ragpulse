ragpulse — 入库与检索

入库 POST /rag/ingest、检索 POST /rag/retrieve、整条 RAG+回答 POST /rag/qa 须使用同一 dept / kb。 PDF 默认 deepdoc；多模态 LLM 需配置 RAG_PUBLIC_BASE_URL + LLM_VISION_MODEL，见 /docs 与仓库 docs/integrations.md。

① 上传文件入库

默认不勾选。勾选后 chunk 多时可能极慢；可用 RAG_LLM_PRE_SUMMARY_MAX_CHUNKS、RAG_LLM_PRE_SUMMARY_MAX_CHUNK_CHARS、LLM_SUMMARY_MODEL 控制。

长文 PDF 说明：选「仅 deepdoc」时，几十页论文可能需 10～30 分钟+（版面/OCR + 每块文本单独调嵌入 API）。 终端在请求结束前往往不会出现 POST /rag/ingest 日志，属正常现象。可先试「PDF（deepdoc 失败再 pypdf）」或「仅 pypdf」验证链路。
